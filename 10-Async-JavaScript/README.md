JavaScript Asynchronous Programming
JavaScript is a single-threaded language, meaning it executes one operation at a time using a single call stack. However, JavaScript can still handle asynchronous operations such as API calls, timers, and file operations by using the event loop and task queues.
Asynchronous programming allows JavaScript to execute long-running operations without blocking the main thread.
This section covers the following concepts:
    Callbacks
    Callback Hell
    Promises
    then() and catch()
    async/await
    Event Loop
    Callback Queue
    Microtask Queue

Callbacks
A callback is a function passed as an argument to another function and executed after the completion of a specific task.
Characteristics:
    Enables asynchronous behavior
    Used in operations like timers, API requests, and event handling
    Allows functions to run after another function finishes execution
Callbacks are one of the earliest methods used in JavaScript for handling asynchronous operations.

Callback Hell
Callback hell occurs when multiple asynchronous operations are nested inside each other, creating deeply nested callbackfunctions.
Problems with callback hell:
    Code becomes difficult to read
    Hard to debug and maintain
    Reduces code clarity
This issue led to the introduction of Promises to manage asynchronous flows more effectively.

Promises
A Promise is an object that represents the eventual completion or failure of an asynchronous operation.
Promise states:
    Pending – initial state, operation not yet completed
    Fulfilled – operation completed successfully
    Rejected – operation failed
Promises provide a cleaner way to handle asynchronous tasks compared to callbacks.
-then() and catch()
Promises use the following methods to handle results:
-then()
    Executes when the promise is resolved
    Used to process the successful result of an asynchronous operation
-catch()
    Executes when the promise is rejected
    Used for handling errors
These methods allow chaining multiple asynchronous operations in a structured way.

Async / Await
async and await were introduced in modern JavaScript to simplify working with promises.
Characteristics:
    async defines an asynchronous function
    await pauses execution until the promise resolves
    Makes asynchronous code look similar to synchronous code
    Improves readability and maintainability
Async/await internally works on top of Promises.

Event Loop
The Event Loop is the mechanism that allows JavaScript to perform non-blocking asynchronous operations.
Responsibilities of the event loop:
    Monitors the call stack
    Checks the task queues
    Moves tasks from queues to the call stack when it becomes empty
This process allows JavaScript to execute asynchronous code efficiently while remaining single-threaded.

Callback Queue
The Callback Queue (also called the Task Queue) stores callbacks from asynchronous operations such as:
    setTimeout
    setInterval
    DOM events
Once the call stack is empty, the event loop pushes callbacks from the callback queue into the call stack for execution.

Microtask Queue
The Microtask Queue stores tasks generated by:
    Promise resolutions
    queueMicrotask()
    Mutation observers
Important rule:
    The microtask queue has higher priority than the callback queue

Key Takeaways
    JavaScript uses asynchronous programming to handle long-running operations efficiently.
    Callbacks were the initial method for managing asynchronous behavior.
    Promises improved asynchronous code structure.
    async/await provides a more readable way to work with promises.
    The event loop coordinates execution between the call stack and task queues.
    Microtasks are executed before callback queue tasks.

Conclusion
Asynchronous programming is essential for building responsive JavaScript applications. Understanding callbacks, promises, async/await, and the event loop helps developers manage complex asynchronous operations efficiently while avoiding performance bottlenecks and blocking behavior.
